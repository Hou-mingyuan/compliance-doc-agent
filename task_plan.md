## 已做决策

| 决策 | 理由 |
| --- | --- |
| 验收状态采用 intake -> executing -> accepted/escalated | 只有新鲜证据齐全才可 accepted |
| 报告选择 DOCX | 可离线生成并做文本、哈希和逐页渲染验收 |
| 采用明确标注的 DEMO 法规目录 | 没有权威数据授权时不伪造真实法规来源 |
| Mock 只整理确定性工具输出 | findings 真实性不依赖固定模型文本 |
| 所有计划、证据和运行产物均留在本仓库 | 满足修改边界 |

## 遇到的错误

| 错误 | 尝试次数 | 解决方案 |
| --- | ---: | --- |
| 目标整文件输出达到显示上限 | 1 | 按标题和行边界读取两节完整正文 |
| PowerShell 外层拦截 `sed`/`&&`/复杂引号 | 多次不同命令 | 改用 `rg`、独立命令或只读 PowerShell，不重复原失败形式 |
| POI lite 类型与 full schema 示例不同 | 1 | 按实际依赖改用可用 OOXML 类型和可空 getter |
| 捆绑 DOCX renderer 找不到 `soffice` | 1 | 使用仓库内 Portable LibreOffice/Poppler，设置 PATH 与唯一 profile 后成功 |
| LibreOffice 输出 Python prefix 提示 | 1 | PNG/PDF 正常生成且逐页检查通过，判定为非阻断环境提示 |
| Compose JSON 管道输出含 UTF-8 BOM | 1 | 使用 `utf-8-sig` 解码重跑，确认只发布 19070/19071 |
| k6 指标读取采用了错误 schema | 1 | 按 summary-export 顶层字段重读，确认读写 p95 与失败率 |
